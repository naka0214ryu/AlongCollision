# AlongCollision
〇開発期間

約2か月（土日祝、他数日を除く）

（実際の作業日数：25日）


〇開発人数

5人（エンジニア3人、プランナー2人）


〇開発環境

Windows 11/ Unity 6000.4.0f1


〇担当箇所

・PHP/MySQLを用いたリアルタイムランキング機能の構築

・敵、スコアの処理全般

・ランキング・リザルト画面のUI

・タイトルロゴのデザイン

　等々
 

〇操作方法

マウス左クリック：

　プレイヤーをマウスの方向に吹っ飛ばす
 
　長押しでより遠くへ飛ばせる
 
マウス右クリック：

　プレイヤーを姫の位置にワープ
 

〇敵の種類

ゾウ：ぶつかってもあまり吹っ飛びません。
 
ブタ：一定時間で右に大きくズレます。
 
ニワトリ：左にちまちまとズレつつけます。
 
ウシ：静止状態から時間が経つと、姫に猛突進してきます。他の敵よりダメージが少し大きいです。


〇作品概要

PHPの学習を目的とした授業の一環で制作した、タワーディフェンス要素のあるアクションゲームです。

ゴールのお城へ向かって歩き続けるお姫様を、様々な動物（敵）から守り抜くことが目的です。

プレイヤーが敵にぶつかって倒すだけでなく、吹き飛んだ敵が他の敵に巻き込まれる連鎖が発生すると、獲得スコアが倍増する爽快感を重視しました。

ステージクリア時には、残り体力に応じたボーナススコアが加算されます。

※ゲーム発表会では、PCをローカルサーバーとしてPHP/データベースを稼働させ、リアルタイムランキング機能を実装しました。（現在は稼働を停止しています）


〇制作で意識したこと

短期間の開発でも敵の種類を簡単に増やせるよう、設計を工夫しました。

敵の共通処理を親クラスにまとめ、個別の敵はそれを継承して動かす構造にしています。

これにより、インスペクターのリストに新しい敵Prefabをアタッチするだけで、ゲームの根幹システムを書き換えることなく要素を追加・拡張できる環境を実現しました。


〇苦労した点

開発中、チームリーダーの強いこだわりやゲーム設計に関する意見と、全員の意見が衝突し、進捗が滞りかける局面がありました。

リーダーのこだわりを否定するのではなく、まずは意図を話し合った上で、短い期限内でユーザーが最も楽しめる仕様は何かを客観的な目線で代替案を用意して、リーダーにも納得してもらいました


〇反省点
動き回る姫を守る要素と、おはじきアクションの要素はあまり相性が良くなく、ゲーム性が噛み合っていなかったと感じました。
また、スケジュール管理が甘かったため、デバッグやクオリティアップに充てる時間を十分に確保できませんでした。今後は制作スケジュールに余裕を持たせ、早い段階から調整や検証を進められるよう改善したいです。
